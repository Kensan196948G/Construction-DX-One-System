## 1. はじめに

### 1.1 本マニュアルの対象者

本マニュアルは、変更申請（RFC）を起票・管理する一般利用者を対象としています。

### 1.2 システムアクセス

| 項目 | 内容 |
|------|------|
| URL | https://cab.mirai-kensetsu.co.jp |
| 対応ブラウザ | Google Chrome（最新版）、Microsoft Edge（最新版）、Firefox（最新版） |
| 認証方式 | 社内ID / パスワード（Entra ID連携） |

## 2. ログイン・ログアウト

### 2.1 ログイン手順

1. ブラウザで `https://cab.mirai-kensetsu.co.jp` にアクセスする
2. ログイン画面が表示される
3. 社内ID（メールアドレス）とパスワードを入力する
4. 「ログイン」ボタンをクリックする
5. ダッシュボード画面が表示されればログイン完了

**ログインできない場合:**

| 状況 | 対処法 |
|------|--------|
| パスワードを忘れた | IT部門にパスワードリセットを依頼 |
| アカウントがロックされた | 5回連続失敗でロック、30分後に自動解除 |
| 画面が表示されない | ブラウザのキャッシュクリア、別ブラウザで試行 |

### 2.2 ログアウト手順

1. 画面右上のユーザーアイコンをクリック
2. 「ログアウト」を選択
3. ログイン画面に遷移すればログアウト完了

---

## 3. ダッシュボード

ログイン後に表示されるダッシュボード画面の各エリアの説明です。

| エリア | 表示内容 |
|--------|---------|
| 自分のRFC一覧 | 自分が起票したRFCの最新一覧（ステータス付き） |
| アクション必要 | 承認待ち、追加情報要求などの対応が必要なRFC |
| 最近の活動 | システム全体の最近の変更活動ログ |
| 統計サマリー | 今月のRFC件数、承認率、進行中の変更数 |
| お知らせ | システムメンテナンス通知、フリーズ期間情報 |

## 4. RFC（変更申請）の起票

### 4.1 RFC起票画面への遷移

1. ダッシュボードの「新規RFC作成」ボタンをクリック
2. または、上部メニュー「RFC管理」→「新規作成」を選択

### 4.2 RFC起票フォーム入力

#### 基本情報

| 入力項目 | 必須 | 説明 | 入力例 |
|---------|------|------|--------|
| タイトル | 必須 | 変更内容を簡潔に記載 | 「ADパスワードポリシー変更（最小8文字→12文字）」 |
| 説明 | 必須 | 変更の目的・内容を詳細に記載 | 変更の背景、実施内容、期待される効果 |
| 変更種別 | 必須 | 標準変更/通常変更/重大変更/緊急変更 | 「通常変更」 |
| 優先度 | 任意 | 低/中/高/緊急 | 「中」 |

#### 対象システム

| 入力項目 | 必須 | 説明 |
|---------|------|------|
| 対象システム | 必須 | 変更対象のシステムを選択（複数選択可） |
| 影響ユーザー数 | 任意 | 影響を受けるユーザーの推定数 |

**対象システム選択肢:**
- Active Directory
- Entra ID
- Exchange Online / Microsoft 365
- HENGEONE
- ファイルサーバー
- DeskNet'sNeo
- AppSuite
- ネットワーク機器
- Azure リソース

#### 日程

| 入力項目 | 必須 | 説明 | 入力例 |
|---------|------|------|--------|
| 実施予定開始日時 | 必須 | 変更作業の開始予定日時 | 2026-04-15 22:00 |
| 実施予定終了日時 | 必須 | 変更作業の完了予定日時 | 2026-04-16 02:00 |

#### 影響分析

| 入力項目 | 必須 | 説明 |
|---------|------|------|
| 業務への影響 | 必須 | 業務にどのような影響があるか |
| 技術リスク | 必須 | 低/中/高/重大から選択 |
| リスク評価 | 必須 | 想定されるリスクと対策 |

**対象システム選択後、「影響分析を実行」ボタンをクリックすると、依存システムとリスクレベルが自動計算されます。**

#### ロールバック計画

| 入力項目 | 必須 | 説明 |
|---------|------|------|
| ロールバック手順 | 必須（通常/重大変更） | 変更失敗時の切り戻し手順 |
| ロールバック判断基準 | 任意 | ロールバックを実行する判断条件 |

### 4.3 下書き保存

- 入力途中で「下書き保存」ボタンをクリックすると、入力内容が保存されます
- 下書きはダッシュボードの「自分のRFC一覧」から再編集できます
- 下書きは提出されるまでCABには表示されません

### 4.4 RFC提出

1. 全ての必須項目を入力完了
2. 「提出」ボタンをクリック
3. 確認ダイアログで内容を確認
4. 「提出する」をクリック
5. RFCステータスが「提出済み」に変更される
6. CABメンバーに審議依頼通知が自動送信される

**注意:** 提出後はRFCの編集ができなくなります。修正が必要な場合は、CABから「差し戻し」を受けた後に編集してください。

## 5. RFC状態の確認

### 5.1 ステータス一覧

| ステータス | 説明 | 次のアクション |
|-----------|------|-------------|
| 下書き（draft） | RFC作成中、未提出 | 編集・提出 |
| 提出済み（submitted） | CABへの審議依頼済み | 承認待ち |
| CAB審議中（cab_review） | CABが審議中 | 審議結果待ち |
| 承認済み（approved） | CABが承認 | 実施予定日に実施 |
| 却下（rejected） | CABが却下 | 修正して再申請 or 取り消し |
| 実施中（in_progress） | 変更作業実施中 | 完了報告 |
| 完了（completed） | 変更作業完了 | PIR（事後レビュー） |
| ロールバック済み（rolled_back） | 変更を切り戻し | PIR（事後レビュー） |
| 取り消し（cancelled） | RFC取り消し | - |

### 5.2 RFC詳細画面

RFC一覧から対象RFCをクリックすると、詳細画面が表示されます。

| タブ | 表示内容 |
|------|---------|
| 基本情報 | RFC内容、対象システム、日程、影響分析 |
| 承認状況 | CABの審議状況、投票結果、承認条件 |
| 実施記録 | 実施チェックリスト、実施結果 |
| 履歴 | ステータス変更履歴、コメント |
| 関連RFC | 同一システム・同一期間の関連RFC |

## 6. 衝突検知アラートへの対応

RFC起票時に衝突が検知された場合、以下のアラートが表示されます。

| アラート種別 | 表示 | 対応方法 |
|------------|------|---------|
| 同一システム重複変更 | 赤色警告「同一システムへの変更が重複しています」 | 実施日程の調整、または関連RFCの担当者と調整 |
| フリーズ期間衝突 | 赤色ブロック「フリーズ期間中です」 | 実施日程をフリーズ期間外に変更 |
| 依存システム変更 | 黄色警告「依存システムに変更が予定されています」 | 関連RFCの内容を確認し、影響を評価 |

## 7. 変更カレンダーの操作

### 7.1 カレンダー画面

上部メニュー「変更カレンダー」をクリックするとカレンダーが表示されます。

| 操作 | 方法 |
|------|------|
| 月表示/週表示/日表示の切替 | カレンダー上部のボタンで切替 |
| 前月/翌月への移動 | 矢印ボタンまたはキーボードの矢印キー |
| 今日に戻る | 「今日」ボタンをクリック |
| 変更の詳細表示 | カレンダー上のイベントをクリック |

### 7.2 カレンダーの色分け

| 色 | 意味 |
|----|------|
| 青 | 通常変更（承認済み・実施予定） |
| 緑 | 標準変更 |
| 赤 | 緊急変更 |
| オレンジ | 重大変更 |
| グレー | 完了済み変更 |
| 赤背景（帯） | フリーズ期間 |

## 9. 通知の確認

### 9.1 通知の受信方法

| 通知チャネル | 設定方法 |
|------------|---------|

| 通知種別 | タイミング | 内容 |
|---------|-----------|------|

| リマインダー | 実施日の前日 | 実施予定のリマインダー |

## 10. よくある質問（FAQ）

| 質問 | 回答 |
|------|------|
| RFCの提出後に修正したい | CABから「差し戻し」を受けるか、一度取り消して再起票してください |
| フリーズ期間中に緊急の変更がある | 変更種別「緊急変更」で起票すれば、フリーズ期間中でも申請可能です |
| 承認にどのくらい時間がかかる | 通常変更は次回CAB会議（週次）で審議されます。目安：3-5営業日 |
| 過去のRFCを検索したい | RFC一覧画面のフィルタ・検索機能をご利用ください |
| 変更のロールバックが必要になった | RFC詳細画面の「ロールバック」ボタンから実行してください |
