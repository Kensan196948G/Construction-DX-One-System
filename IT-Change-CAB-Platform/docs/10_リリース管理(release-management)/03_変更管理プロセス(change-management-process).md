## 1. 概要

### 1.1 目的

本文書は、IT変更管理・リリース自動化プラットフォーム（以下「本プラットフォーム」）自体の変更管理プロセスを定義する。本プラットフォームは変更管理ツールとして他システムの変更を管理するが、自らの変更も同じプロセスに準拠して管理する（自己準拠）。

### 1.2 自己準拠の原則

| 原則 | 内容 |
|------|------|
| 自己適用 | 本プラットフォームの変更は、本プラットフォーム自身のCABプロセスで承認する |
| ISO20000準拠 | 全ての変更がISO20000変更管理プロセスに準拠 |
| 証跡管理 | 変更の記録・承認履歴を本プラットフォーム内に保持 |
| フリーズ期間遵守 | 本プラットフォーム自身もフリーズ期間中の変更を制限 |

## 2. 変更種別と適用基準

### 2.1 本プラットフォームの変更種別

| 変更種別 | 適用条件 | 承認フロー | 例 |
|---------|---------|-----------|-----|
| 標準変更 | 事前承認済みの定型変更 | 自動承認 | 依存パッケージのパッチ更新、ログレベル変更 |
| 通常変更 | 一般的な機能追加・修正 | CAB承認 | 新機能追加、UI改善、バグ修正 |
| 重大変更 | 基盤影響・破壊的変更 | CAB承認 + 経営承認 | DBスキーマ変更、API破壊的変更、インフラ変更 |
| 緊急変更 | セキュリティ脆弱性・重大バグ | ECAB承認 | 脆弱性パッチ、データ不整合修正 |

### 2.2 変更影響度判定マトリクス

| 影響項目 | 低 | 中 | 高 |
|---------|---|---|---|
| ユーザー影響 | UI微調整 | 機能変更（後方互換あり） | 機能変更（後方互換なし） |
| データ影響 | なし | データ移行あり（可逆） | スキーマ破壊的変更 |
| 可用性影響 | ダウンタイムなし | 短時間停止（5分以内） | 長時間停止（5分超） |
| セキュリティ影響 | なし | 認可ルール変更 | 認証方式変更 |

## 3. 変更管理プロセスフロー

### 3.1 通常変更プロセス

```
1. 変更要求の発生
   │  開発者がGitHub Issueを起票
   │
2. RFC起票（本プラットフォーム上）
   │  ├── 変更種別: 通常変更
   │  ├── 対象システム: IT-Change-CAB-Platform
   │  ├── 影響分析: 影響するコンポーネント・データ
   │  ├── テスト計画: テスト項目・テスト環境
   │  └── ロールバック計画: 手順・判断基準
   │
3. 開発・テスト
   │  ├── featureブランチで開発
   │  ├── 単体テスト・結合テスト実行
   │  ├── プルリクエスト作成
   │  └── コードレビュー
   │
4. CAB審議・承認
   │  ├── 週次CAB会議で審議
   │  ├── テスト結果の報告
   │  └── 承認/却下/差し戻し
   │
5. ステージングデプロイ
   │  ├── ステージング環境にデプロイ
   │  └── E2Eテスト・受入テスト
   │
6. 本番デプロイ
   │  ├── デプロイ手順に従い実施
   │  ├── スモークテスト
   │  └── 完了通知
   │
7. 事後レビュー（PIR）
      ├── デプロイ結果の記録
      ├── 問題があれば教訓を記録
      └── RFCクローズ
```

### 3.2 緊急変更プロセス

```
1. 緊急事象の検知
   │  セキュリティ脆弱性 / 重大バグ / サービス障害
   │
2. 緊急RFC起票
   │  ├── 変更種別: 緊急変更
   │  ├── 関連インシデントID
   │  └── 最小限の影響分析
   │
3. ECAB緊急承認
   │  ├── IT部門長（CAB議長）の電話/Teams承認
   │  └── 承認記録を事後登録
   │
4. 緊急修正・デプロイ
   │  ├── hotfixブランチで修正
   │  ├── 最低限のテスト実施
   │  └── 本番デプロイ
   │
5. 事後対応
      ├── 正式なRFC記録の補完
      ├── 詳細テストの実施
      ├── 根本原因分析（RCA）
      └── 再発防止策の策定
```

## 4. 役割と責任

### 4.1 RACI マトリクス

| 活動 | 開発者 | レビュアー | CABメンバー | CAB議長 | 運用チーム |
|------|--------|-----------|-----------|---------|-----------|
| RFC起票 | R | I | I | I | I |
| 影響分析 | R | C | I | I | C |
| コード実装 | R | I | - | - | - |
| コードレビュー | I | R | - | - | - |
| テスト実行 | R | C | - | - | C |
| CAB審議 | I | I | R | A | I |
| デプロイ実施 | C | - | I | I | R |
| スモークテスト | C | - | - | - | R |
| 事後レビュー | R | C | I | A | C |

R: 実行責任 / A: 承認責任 / C: 相談先 / I: 情報共有先

## 5. 変更記録管理

### 5.1 記録項目

| 項目 | 内容 |
|------|------|
| RFC番号 | 本プラットフォーム内で自動採番 |
| GitHub Issue番号 | 開発管理用Issue番号 |
| GitHub PR番号 | プルリクエスト番号 |
| Git タグ | リリースバージョンタグ |
| 変更内容 | 変更の詳細説明 |
| 影響分析結果 | 影響範囲・リスク評価 |
| テスト結果 | テスト合格状況 |
| CAB承認記録 | 承認者・承認日時・条件 |
| デプロイ記録 | デプロイ日時・実施者・結果 |
| PIR記録 | 事後レビュー結果 |

### 5.2 トレーサビリティ

```
GitHub Issue  ←→  RFC（本プラットフォーム）  ←→  GitHub PR  ←→  Git Tag
    │                    │                        │              │
    │                    │                        │              │
    ▼                    ▼                        ▼              ▼
 要件定義          承認記録・証跡          コード変更履歴     リリース版
```

## 6. 変更管理KPI

### 6.1 自己変更管理のKPI

| KPI | 目標値 | 測定方法 |
|-----|--------|---------|
| 変更成功率 | 95%以上 | 完了RFC / 全RFC（cancelled除く） |
| ロールバック率 | 5%以下 | ロールバックRFC / デプロイRFC |
| 平均承認リードタイム | 3営業日以内 | RFC提出～承認の平均日数 |
| 緊急変更比率 | 10%以下 | 緊急RFC / 全RFC |
| 変更起因インシデント率 | 5%以下 | インシデント紐付きRFC / 全RFC |
| テストカバレッジ | 80%以上 | Istanbul計測 |

### 6.2 KPIレビュー

| レビュー | 頻度 | 参加者 |
|---------|------|--------|
| 月次KPIレビュー | 毎月月初 | 開発チーム、運用チーム |
| 四半期改善レビュー | 四半期末 | IT部門全体 |
| 年次プロセスレビュー | 年度末 | IT部門 + 経営層 |

## 7. コンプライアンスとの整合

### 7.1 ISO20000準拠項目

| ISO20000要件 | 本プロセスでの対応 |
|-------------|-----------------|
| 変更の計画と管理 | RFC起票による変更計画の文書化 |
| 変更の評価とリスク分析 | 影響分析エンジンによる自動評価 |
| 変更の承認 | CAB/ECAB承認プロセス |
| 変更の記録と追跡 | 全変更のRFC記録、ステータス追跡 |
| 変更のレビュー | PIR（事後レビュー）プロセス |
| 変更方針と手順の文書化 | 本文書および関連ドキュメント |

### 7.2 監査対応

| 監査項目 | 証跡 | 保管場所 |
|---------|------|---------|
| 変更承認記録 | CAB議事録、承認メール | 本プラットフォーム内 |
| 変更実施記録 | デプロイログ、スモークテスト結果 | 本プラットフォーム内 + Git |
| テスト記録 | CI/CDパイプラインログ | GitHub Actions |
| ロールバック記録 | ロールバックRFC、障害報告書 | 本プラットフォーム内 |
