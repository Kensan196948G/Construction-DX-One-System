## 8. フェーズ間依存関係

```
Phase 0（計画・準備）
  │
  ├─ 要件定義書 ──────────→ Phase 1, 2, 3, 4 全てに影響
  ├─ 設計仕様書 ──────────→ Phase 1, 2 の実装に必要
  └─ 技術POC ────────────→ Phase 1 の技術選択に影響
       │
Phase 1（基盤構築）
  │
  ├─ 認証基盤 ───────────→ Phase 2 全画面で利用
  ├─ DB スキーマ ─────────→ Phase 2, 3 のデータ操作に必須
  ├─ API 共通基盤 ────────→ Phase 2, 3 の全API実装に必須
  └─ BullMQ 基盤 ─────────→ Phase 2（通知）, Phase 3（Teams連携）
       │
Phase 2（コア機能）
  │
  ├─ RFC 管理 ───────────→ Phase 3 の影響分析・KPIの前提
  ├─ CAB ワークフロー ────→ Phase 3 のPIR・Teams通知の前提
  └─ 変更カレンダー ──────→ Phase 3 のトレンド分析に利用
       │
Phase 3（高度機能）
  │
  ├─ 全機能完成 ─────────→ Phase 4 のUAT対象
  └─ 性能テスト結果 ──────→ Phase 4 の性能チューニング入力
       │
Phase 4（運用最適化）
  │
  └─ 本番リリース ────────→ 安定化・ISO20000監査対応
```
