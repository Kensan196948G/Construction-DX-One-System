## 3. クリティカルパス

以下がプロジェクトのクリティカルパスである。これらのタスクに遅延が生じると、プロジェクト全体のスケジュールに直接影響する。

```
要件定義確定(2.1.2)
  → DB設計(2.2.2)
    → DBテーブル実装(3.1.3)
      → RFC CRUD API(4.1.1)
        → RFC状態遷移(4.1.2)
          → 承認ワークフロー(4.2.2)
            → 結合テスト(4.6.1)
              → MVPリリース(4.6.4)
                → KPI集計API(5.2.1)
                  → 性能テスト(5.5.3)
                    → 性能最適化(6.1.1)
                      → UAT(6.4.3)
                        → 本番リリース(6.5.3)
```
