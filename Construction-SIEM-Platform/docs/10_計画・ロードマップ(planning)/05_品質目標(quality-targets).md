### 複雑度基準（C901）

| 複雑度 | 判定 | アクション |
|:------:|------|-----------|
| 1-5 | 🟢 良好 | そのまま |
| 6-10 | 🟡 許容 | リファクタリング検討 |
| 11-15 | 🟠 要改善 | リファクタリング必須 |
| 16+ | 🔴 不合格 | CI失敗（分割必須） |

### コーディング規約

| インデント | スペース4つ |

| docstring | Google Style |

### MTTD / MTTR 目標

| 指標 | 定義 | 目標 | 計測方法 |
|------|------|:----:|---------|
| **MTTD** | イベント発生〜検知 | **15分以内** | テストイベント注入〜アラート生成 |
| **MTTR** | 検知〜復旧完了 | **2時間以内** | アラート生成〜インシデントクローズ |
| **MTTC** | 検知〜封じ込め | **1時間以内（P1）** | アラート生成〜封じ込め完了 |

```
目標タイムライン（P1インシデント）:

イベント発生 ─── 15分 ──► 検知 ─── 15分 ──► 初動
                                              │
                          ◄── 30分 ──── 封じ込め
                          │
                          ◄── 2時間 ──── 復旧完了
```

### 測定頻度

| 指標 | 測定頻度 | 測定方法 |
|------|:-------:|---------|
| テスト数 | 毎コミット | pytest --co -q \| wc -l |
| カバレッジ | 毎コミット | pytest --cov |
| bandit | 毎コミット | CI自動実行 |
| C901 | 毎コミット | flake8 --select=C901 |
| CI成功率 | 週次集計 | GitHub Actions統計 |
| MTTD/MTTR | 月次計測 | テストシナリオ実行 |
| コンプライアンス | 週次 | API自動チェック |
| EPS | Phase 20 | 負荷テスト |

### 品質レポート出力

```bash
# 品質サマリー取得
echo "=== Quality Report ==="
echo "Tests: $(pytest --co -q 2>/dev/null | tail -1)"
echo "Coverage: $(pytest --cov --cov-report=term-missing 2>/dev/null | grep TOTAL)"
echo "Bandit: $(bandit -r src/ -f json 2>/dev/null | python3 -c 'import sys,json; print(json.load(sys.stdin)["metrics"]["_totals"]["SEVERITY.HIGH"])')"
echo "Flake8 C901: $(flake8 --select=C901 . 2>/dev/null | wc -l)"
```
