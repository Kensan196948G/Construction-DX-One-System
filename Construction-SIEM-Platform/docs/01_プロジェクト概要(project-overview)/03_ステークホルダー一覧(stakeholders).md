## 🛡 CISO（最高情報セキュリティ責任者）

| 項目 | 内容 |
|------|------|
| **責務** | セキュリティ戦略の立案・承認、ガバナンス統括、コンプライアンス最終責任 |
| **関心事** | リスク状況の全体把握、KPI 達成状況、規格準拠状況、インシデント傾向 |
| **意思決定** | セキュリティポリシー策定、重大インシデント対応方針、予算配分 |

### 🖥 利用機能

| 機能 | エンドポイント例 | 用途 |
|------|-----------------|------|
| 📊 KPI ダッシュボード | `GET /api/v1/kpi/dashboard` | MTTD/MTTR/誤検知率の確認 |
| 📋 コンプライアンスレポート | `GET /api/v1/compliance/check` | ISO 27001/NIST CSF 準拠チェック |
| 📈 インシデント統計 | `GET /api/v1/incidents/statistics` | インシデント傾向分析 |
| 📄 報告書生成 | `GET /api/v1/reports/generate` | 経営層向けレポート |
| 🔍 監査ログ閲覧 | `GET /api/v1/audit/logs` | 操作履歴の確認 |

---

## 💻 IT 管理者

| 項目 | 内容 |
|------|------|
| **責務** | システム運用管理、インフラ監視、ユーザー管理、バックアップ・復旧 |
| **関心事** | システム稼働率、パフォーマンス、ストレージ容量、アラート管理 |
| **意思決定** | システム構成変更、スケーリング判断、メンテナンス計画 |

### 🖥 利用機能

| 機能 | エンドポイント例 | 用途 |
|------|-----------------|------|
| 🔐 ユーザー管理 | `POST /api/v1/auth/register` | アカウント作成・権限管理 |
| 📊 システムメトリクス | `GET /api/v1/metrics` | Prometheus メトリクス確認 |
| ⚙️ ヘルスチェック | `GET /api/v1/health` | 各コンポーネント稼働状況 |
| 🔔 通知設定 | `POST /api/v1/notifications/config` | アラート通知先設定 |
| 📦 Elasticsearch 管理 | `GET /api/v1/elasticsearch/health` | インデックス・ILM 管理 |

## 🔍 SOC アナリスト

| 項目 | 内容 |
|------|------|
| **責務** | インシデント監視・調査、アラートトリアージ、対応プレイブック実行、脅威ハンティング |
| **関心事** | リアルタイムアラート、イベント相関分析、IoC マッチ、インシデント管理 |
| **意思決定** | アラート優先度判断、エスカレーション判断、対応手順選択 |

### 🖥 利用機能

| 機能 | エンドポイント例 | 用途 |
|------|-----------------|------|
| 🚨 アラート管理 | `GET /api/v1/alerts` | アラート一覧・フィルタリング |
| 📝 インシデント管理 | `POST /api/v1/incidents` | インシデント起票・更新 |
| 📖 プレイブック実行 | `POST /api/v1/playbooks/execute` | 自動対応手順の実行 |
| 🕵️ 脅威インテリジェンス | `GET /api/v1/threat-intel/ioc` | IoC 照合・検索 |
| 🔗 相関分析 | `GET /api/v1/correlation/results` | イベント相関・キルチェーン分析 |
| 🔍 イベント検索 | `GET /api/v1/events/search` | ログ検索・フィルタリング |
| 📊 Grafana ダッシュボード | — | リアルタイム監視 |
| 📈 Kibana | — | ログ詳細分析・可視化 |

## 🏗 現場管理者

| 項目 | 内容 |
|------|------|

## 🗺 RACI マトリクス

| 活動 | CISO | IT 管理者 | SOC アナリスト | 現場管理者 |
|------|:----:|:---------:|:-------------:|:---------:|
| セキュリティポリシー策定 | **A** | C | C | I |
| システム運用管理 | I | **A/R** | C | I |
| インシデント監視 | I | C | **A/R** | I |
| インシデント対応 | I | C | **R** | C |
| エスカレーション判断 | **A** | C | **R** | I |
| 現場機器一次対応 | I | C | C | **A/R** |
| コンプライアンス監査 | **A** | **R** | C | I |
| レポート承認 | **A** | R | C | I |

> **R** = Responsible（実行責任）, **A** = Accountable（説明責任）, **C** = Consulted（相談）, **I** = Informed（報告）
