## ⚡ パフォーマンス要件

### NFR-PERF-001: イベント処理スループット

| 項目 | 内容 |
|------|------|
| **要件ID** | NFR-PERF-001 |
| **要件名** | イベント処理スループット |
| **目標値** | 🎯 **10,000 EPS（Events Per Second）以上** |
| **説明** | Kafka パイプライン経由のイベント処理能力。正規化・ルール評価・格納までの一連の処理 |
| **測定方法** | Prometheus メトリクス `siem_events_processed_total` のレート |
| **現状** | ✅ 達成 |

### NFR-PERF-002: アラート生成遅延

| 項目 | 内容 |
|------|------|
| **要件ID** | NFR-PERF-002 |
| **要件名** | アラート生成遅延 |
| **目標値** | 🎯 **イベント検知から3分以内にアラート生成** |
| **説明** | 脅威イベントが発生してからアラートが生成・通知されるまでの遅延 |
| **測定方法** | `siem_alert_generation_latency_seconds` |
| **現状** | ✅ 達成 |

### NFR-PERF-003: 検索応答時間

| 項目 | 内容 |
|------|------|
| **要件ID** | NFR-PERF-003 |
| **要件名** | ログ検索応答時間 |
| **目標値** | 🎯 **30秒以内**（直近7日間のデータ） |
| **説明** | Elasticsearch に対するログ検索クエリの応答時間 |
| **測定方法** | API レスポンスタイム `GET /api/v1/events/search` |
| **現状** | ✅ 達成 |

### NFR-PERF-004: API レスポンスタイム

| 項目 | 内容 |
|------|------|
| **要件ID** | NFR-PERF-004 |
| **要件名** | API レスポンスタイム |
| **目標値** | 🎯 **95パーセンタイルで500ms以内** |
| **説明** | 全 API エンドポイントの一般的なレスポンスタイム |
| **測定方法** | Prometheus ヒストグラム `http_request_duration_seconds` |
| **現状** | ✅ 達成 |
