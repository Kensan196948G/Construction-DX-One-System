# 📊 SLA 定義書

> Construction-SIEM-Platform のサービスレベル合意（SLA）を定義する

---

## 🎯 SLA 概要

本文書は、Construction-SIEM-Platform が提供するセキュリティ監視サービスのサービスレベル目標を定義する。ISO 20000 に準拠し、インシデント優先度に応じた応答・解決時間を規定する。

---

## 🚨 インシデント優先度分類

| 優先度 | 名称 | 定義 | 例 |
|:------:|------|------|-----|
| 🔴 **P1** | 緊急 | ビジネスクリティカルなセキュリティインシデント。即座の対応が必要 | ランサムウェア感染、データ漏洩、APT 攻撃 |
| 🟠 **P2** | 高 | 重大なセキュリティイベント。迅速な対応が必要 | ブルートフォース攻撃、不正アクセス試行、IoT デバイス侵害 |
| 🟡 **P3** | 中 | 調査が必要なセキュリティイベント | 異常な UEBA スコア、ポリシー違反、設定ミス |
| 🟢 **P4** | 低 | 情報レベルのセキュリティイベント | ログ収集警告、パフォーマンス低下、定期レポート異常 |

---

## ⏱ 応答・解決時間 SLA

### 📋 SLA マトリクス

| 優先度 | 応答時間 | 解決時間 | エスカレーション | 通知チャネル |
|:------:|:--------:|:--------:|:----------------:|:------------:|
| 🔴 **P1** | ⏱ **15分** | ⏱ **2時間** | 30分後 → CISO | 📧 メール + 💬 Slack + 📞 電話 |
| 🟠 **P2** | ⏱ **30分** | ⏱ **8時間** | 2時間後 → IT管理者 | 📧 メール + 💬 Slack |
| 🟡 **P3** | ⏱ **2時間** | ⏱ **24時間** | 8時間後 → チームリーダー | 📧 メール |
| 🟢 **P4** | ⏱ **8時間** | ⏱ **72時間** | 24時間後 → チームリーダー | 📧 メール |

### 📝 定義

| 用語 | 定義 |
|------|------|
| **応答時間** | インシデント検知から SOC アナリストが確認（acknowledged）するまでの時間 |
| **解決時間** | インシデント検知から対応完了（resolved）するまでの時間 |
| **エスカレーション** | 応答時間超過時の自動エスカレーション先 |

## 📈 KPI 目標

### 🎯 主要 KPI

| KPI | 目標値 | 測定方法 | 報告頻度 |
|-----|:------:|----------|:--------:|
| 🔍 **MTTD**（平均検知時間） | **< 15分** | `siem_mttd_seconds` メトリクス | 日次 |
| 🛡 **MTTR**（平均対応時間） | **< 2時間** | `siem_mttr_seconds` メトリクス | 日次 |
| ⚠️ **誤検知率** | **< 5%** | 誤検知アラート数 / 全アラート数 | 週次 |
| 📊 **検知カバレッジ** | **100%** | 監視対象デバイス数 / 全デバイス数 | 月次 |

### 📊 補助 KPI

| KPI | 目標値 | 説明 |
|-----|:------:|------|
| 📥 イベント処理スループット | ≥ 10,000 EPS | パイプラインの処理能力 |
| 🟢 システム稼働率 | ≥ 99.9% | コアコンポーネントの可用性 |
| 📋 SLA 達成率 | ≥ 95% | 各優先度の SLA 達成割合 |
| 🔄 プレイブック成功率 | ≥ 90% | 自動プレイブックの実行成功率 |
| 📝 インシデント完了率 | ≥ 98% | 期限内にクローズされたインシデント割合 |

## 🔄 エスカレーションフロー

### P1（緊急）エスカレーション

```
[0分]   アラート検知 → SOC アナリストに自動通知
[15分]  応答なし → SOC チームリーダーにエスカレーション
[30分]  応答なし → IT 管理者 + CISO にエスカレーション
[60分]  未解決 → 全関係者への緊急招集
[2時間] SLA 超過 → SLA 違反レポート生成
```

### P2（高）エスカレーション

```
[0分]   アラート検知 → SOC アナリストに自動通知
[30分]  応答なし → SOC チームリーダーにエスカレーション
[2時間] 応答なし → IT 管理者にエスカレーション
[8時間] SLA 超過 → SLA 違反レポート生成
```

### P3/P4 エスカレーション

```
[応答時間超過] → SOC チームリーダーにエスカレーション
[解決時間超過] → SLA 違反レポート生成
```

## 📊 SLA 測定・報告

### 📅 報告スケジュール

| レポート | 頻度 | 対象者 | 内容 |
|----------|:----:|--------|------|
| 📊 日次レポート | 毎日 | SOC チーム | MTTD/MTTR、アラート統計、SLA 達成率 |
| 📈 週次レポート | 毎週月曜 | IT 管理者 | 週間トレンド、誤検知分析、改善提案 |
| 📋 月次レポート | 毎月1日 | CISO | 月間 KPI、SLA 達成率、コンプライアンス状況 |
| 📄 四半期レポート | 四半期末 | 経営層 | 長期トレンド、投資効果分析、次期計画 |

### 🖥 測定ツール

| ツール | 用途 |
|--------|------|
| 📊 Grafana | リアルタイム KPI ダッシュボード |
| 📈 Prometheus | メトリクス収集・保存 |
| 🔌 KPI API | `GET /api/v1/kpi/dashboard` による KPI 取得 |
| 📋 レポート API | `GET /api/v1/reports/generate` によるレポート生成 |

## ⚠️ SLA 違反時の対応

### 是正プロセス

| ステップ | 内容 | 担当 |
|:--------:|------|------|
| 1️⃣ | SLA 違反の検知・記録 | 自動 |
| 2️⃣ | 根本原因分析（RCA） | SOC チームリーダー |
| 3️⃣ | 是正措置の策定 | IT 管理者 |
| 4️⃣ | 是正措置の実施 | 担当チーム |
| 5️⃣ | 効果確認・レポート | CISO |

### ペナルティ基準

| 違反レベル | 条件 | 対応 |
|:----------:|------|------|
| ⚠️ 注意 | 月間 SLA 達成率 90-95% | 改善計画提出 |
| 🟠 警告 | 月間 SLA 達成率 80-90% | 緊急改善計画 + 週次レビュー |
| 🔴 重大 | 月間 SLA 達成率 < 80% | 体制見直し + 是正措置報告 |

| 項目 | 内容 |
